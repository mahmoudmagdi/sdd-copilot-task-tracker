
# Demo Roadmap

This project demonstrates spec-driven development with GitHub Copilot.

## Demo Flow

1. Create a project from scratch.

2. Add specs before implementation.

3. Add Copilot instructions.

4. Create GitHub issues from implementation tasks.

5. Assign a GitHub issue to Copilot.

6. Let Copilot create a PR.

7. Review the PR against the spec.

8. Merge only after human review and CI validation.

## Important Demo Message

Copilot is not replacing engineering review.

The correct workflow is:

Issue -> Spec -> Copilot implementation -> Pull Request -> CI -> Human Review -> Merge
